Color Remapping
for Edge-Matching Puzzles

A systematic framework for solving edge-matching puzzles by reducing the color space through binary — or generalised k-color — remappings. Each remapping transforms the original puzzle into a simpler variant whose solutions contain all solutions of the original. Intersecting solution sets across a discriminating family of remappings recovers the exact solution set.

This collection covers the theoretical foundations, a worked 3×3 example with full solution synthesis, and an interactive explorer for the generalised k-color type complexity.

Key concepts
Binary remapping
f : Γ → {0,1,2} · maps non-boundary colors to two groups · 2c possible remappings for c colors
Canonical type
lexicographically minimal rotation of (N,E,S,W) · 18 types for k=2 · T(k) = k(k+1)²(k+2)/4 in general
Discriminating family
set of remappings such that ∩ Sol(Pfᵢ) = Sol(P) · minimum size ⌈logT(k)(N)⌉
Type inventory
T(k)-dimensional vector counting pieces per canonical type · drives DP row enumeration
Piece-index solutions
∏ nₜ! assignments per type solution · expanded on demand after intersection
Rotation phantoms
type-pair consistent but physically unrealisable — arise when composed original-piece rotations differ across remappings
Row generation + DP
enumerate valid type-inventory row vectors · multiset DP finds combinations summing to puzzle inventory
Threading
row-by-row assembly via boundary key matching · prunes incompatible rows early · scales linearly with distributed solvers
